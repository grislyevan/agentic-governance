# Detec Polished Collateral — Squarespace Upload Guide

This folder contains publication-ready collateral converted from the current markdown sources.

## Files
- `one-sheet.html`
- `whitepaper.html`
- `one-sheet.txt`
- `whitepaper.txt`

## Recommended Squarespace setup

### 1) One-sheet page
Create a standard page (not a blog post) named **Detec One-Sheet**.

Suggested section order:
1. Hero: tagline + descriptor
2. Problem
3. Tool Classes table
4. Detection model (five signal layers)
5. Enforcement ladder
6. Audit and evidence
7. Known limits
8. Pilot CTA

CTA button links to add:
- **Start a Pilot** → your contact form or scheduling page
- **Download Whitepaper** → link to `whitepaper.html` page or a hosted PDF if you generate one later

### 2) Whitepaper page
Create a long-form page named **Detec Whitepaper**.

Suggested section order:
1. Executive Summary
2. Governance Gap
3. Detection Model
4. Confidence Scoring
5. Policy and Enforcement
6. Architecture
7. Validation
8. Known Limits
9. Deployment
10. Conclusion + contact

CTA button links to add:
- **Request Technical Review** → contact form
- **Start Pilot** → pilot intake form

## Styling notes (match Detec brand)
- Typography:
  - Primary: IBM Plex Sans (400, 500, 600, 700)
  - Technical labels/snippets: IBM Plex Mono (400, 500)
- Color semantics:
  - Primary (soft indigo): `#6366f1`
  - Secondary (teal): `#14b8a6`
  - Attention (amber): `#f59e0b`
  - Neutral text/background: Slate scale (`#0f172a` to `#f8fafc`)
- Voice and copy tone:
  - Precise, calm, evidence-first
  - Avoid hype claims and fear framing
  - Keep known limits visible

## Upload methods
- If Squarespace code injection/custom HTML blocks are available, paste the HTML body content into a Code Block section.
- If not, paste from the `.txt` versions into native text blocks and recreate tables/lists using Squarespace blocks.
