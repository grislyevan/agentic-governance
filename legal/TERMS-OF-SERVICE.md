## 3. Description of Service

Detec provides endpoint governance for agentic AI tools. The Software consists of three primary components:

**3.1 Detec Agent.** An endpoint agent that runs on macOS, Windows, and Linux. The Agent:
- Scans for agentic AI tools across five signal layers (process, file, network, identity, behavior)
- Computes a confidence score (0.00 to 1.00) for each detected tool using calibrated, per-tool weights
- Classifies tools by capability class (Class A through D)
- Evaluates organizational policy to produce enforcement decisions (Detect, Warn, Approval Required, Block)
- Transmits structured event metadata to the Server via HTTP (REST API) or persistent TCP connection (binary msgpack protocol)

**3.2 Detec Server.** A backend application that:
- Receives and stores canonical detection events from Agents
- Manages endpoint enrollment, heartbeats, and status tracking
- Enforces multi-tenant data isolation via tenant-scoped records
- Provides a RESTful API for event queries, policy management, and user administration
- Supports SQLite (single-node) and PostgreSQL (production) databases
- Optionally operates a binary protocol gateway on a configurable TCP port

**3.3 Detec Dashboard.** A web-based security operations interface that:
- Displays detection events with filtering, search, and time-range controls
- Provides policy management, endpoint inventory, and audit log views
- Supports role-based access (Owner, Admin, Analyst, Viewer)
- Authenticates users via JWT with automatic token refresh

## 7. Data Collection and Privacy

### 7.1 What the Agent Collects Locally

The Agent collects Telemetry Data across five signal layers to compute confidence scores for tool detection. This data is processed locally on the Endpoint and is **not** transmitted to the Server:

- **Process signals:** Running process names, command-line arguments, process trees, process ownership
- **File signals:** File existence checks, directory listings, file sizes and modification times, configuration file contents (for tool identification only)
- **Network signals:** Active network connections, remote addresses and ports, connection counts
- **Identity signals:** Operating system username, git configuration (user.email), credential store presence, code-signing identity
- **Behavior signals:** Recent activity timestamps, transcript or session counts, tool-specific activity markers

### 7.2 What the Agent Transmits

The Agent transmits structured event metadata to the Server. Transmitted data includes:

- **Event metadata:** Event ID, event type, event version, timestamps (observed_at, ingested_at)
- **Correlation identifiers:** Session ID, trace ID, parent event ID
- **Actor identifier:** Configurable; defaults to the operating system username of the logged-in user
- **Endpoint identifier:** Configurable; defaults to the system hostname
- **Endpoint context:** Operating system type, version, and architecture
- **Tool detection:** Tool name, capability class (A/B/C/D), version (when available), confidence score, attribution sources
- **Policy decisions:** Rule ID, decision state (detect/warn/approval_required/block), severity level
- **Enforcement actions:** Enforcement type applied, outcome status

The Agent does **not** transmit:
- Raw process command-line arguments or process trees
- File paths, repository paths, or file contents
- Network connection details (addresses, ports)
- Git configuration values (user.email)
- Credential store contents or authentication tokens
- IDE telemetry or extension data content

### 7.3 What the Server Stores

The Server stores the following categories of data:

- **Events:** Full canonical event payloads as received from Agents, including all fields listed in Section 7.2
- **Endpoints:** Hostname, operating system information, posture data, heartbeat interval, last-seen timestamp, enrollment status, and (optionally) a public signing key and key fingerprint for event verification
- **User accounts:** Email address, first name, last name, hashed password (never plaintext), role, hashed API key, tenant association
- **Audit log:** Actor ID, action performed, resource type and ID, detail payload, IP address, timestamp

### 7.4 Data Security

- API keys are stored as SHA-256 hashes. The raw key is shown once at creation and is not recoverable.
- User passwords are hashed before storage and are never stored or logged in plaintext.
- Event signing using Ed25519 key pairs is available for cryptographic verification of event integrity.
- Multi-tenant deployments enforce row-level tenant isolation on all data tables.
- JWT tokens are short-lived and require a strong, unique secret configured per deployment.

### 7.5 No Third-Party Data Sharing

Detec does not transmit Customer data to any third party. For On-Premises Deployments, all data remains on Customer infrastructure. For the Hosted Service, Detec processes data solely to provide the Service and does not sell, share, or monetize Customer data.
